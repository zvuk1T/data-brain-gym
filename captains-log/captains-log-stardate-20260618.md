## 🧠 KEY CONCEPTS COVERED

| Concept | Lesson | Core principle |
|---------|--------|----------------|
| Why visualizations work | L1 | Brain processes visual patterns faster than raw numbers — Datasaurus: same stats, different shapes |
| Purpose of visualization | L2 | Simplify and highlight insights — not manipulate, not hide errors |
| Three correct visualization outcomes | L3 | Faster decisions · any dataset size · more complete than aggregation alone |
| Bar chart vs histogram | L4 | Same shape, different data type: categorical → bar, numerical → histogram |
| Bar chart flexibility | L5 | Bar charts show any summary per category — count is most common, not the only option |
| Chart selection decision rule | L6 | Data type + number of variables + objective → chart type |
| Stacked vs 100% stacked | L7–L9 | Stacked: absolute breakdown. 100% stacked: relative share. Signal word: "relative" → 100% stacked |
| Three guiding principles | L10 | Trends not values · simplify format and decoration · meet audience expectations |
| Pie chart failure pattern | L10 | Can't compare areas → overlay numbers → reading raw data with extra steps → use bar chart instead |
| Format-first thinking | L12 | Choosing chart before seeing data reverses the correct process — fatal mistake |
